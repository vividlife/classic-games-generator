# TASK-2026-03-22-werewolf-night-info-leak-fix

## 背景
classic-games-generator 项目的狼人杀存在一个夜晚信息泄露 bug。

## 用户反馈
开始游戏后，进入“狼人请睁眼”阶段时，系统已经直接把狼人身份提示出来了。这会破坏游戏流程。该阶段不应该向所有玩家直接暴露狼人是谁。

## 任务目标
修复狼人杀夜晚流程中的信息泄露问题：
- 在“狼人请睁眼”阶段，不能直接展示狼人是谁给全体界面/旁观者。
- 保留合理的游戏引导与流程推进能力。
- 不要破坏现有的夜晚流程、发牌逻辑、白天投票、结算等功能。

## 执行要求
1. 先定位是 UI 文案泄露、状态机泄露，还是角色信息面板被错误复用。
2. 仅做与该 bug 直接相关、低风险的修复；不要顺手做大范围重构。
3. 修复后自行做最小必要验证，至少覆盖：
   - 开始游戏
   - 进入“狼人请睁眼”阶段
   - 确认此时不会直接显示狼人名单/身份给公共界面
   - 基本流程未被修坏（至少能继续推进）
4. 如果项目有 lint / build / 可执行测试，优先跑与本改动相关的最小验证。
