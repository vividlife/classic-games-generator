# TASK-2026-03-23-add-2048-game

## 背景
classic-games-generator 是当前高活跃的经典游戏项目，首页已集成多个小游戏。现在需要尽快新增一个经典的 2048 游戏，并把它接入现有站点。

## 用户需求
- 在这个经典游戏项目里新增一个“2048”经典游戏。
- 要尽快开发完成。
- 需要和当前站点风格保持一致，能从首页进入游玩。

## 任务目标
实现一个可游玩的 2048 页面，并完成基础接入：
- 新增 `/2048`（或等价可接受的稳定路由，优先 `/2048`）游戏页面。
- 首页增加 2048 游戏入口卡片。
- 游戏至少支持：
  - 4x4 经典棋盘
  - 随机生成 2/4
  - 上下左右移动与合并
  - 分数统计
  - 游戏结束判定
  - 重新开始
- 兼顾桌面端与移动端基本可用性。
- 保持现有项目视觉风格统一（暗色卡片、按钮、文案风格等）。

## 执行要求
1. 先阅读并理解现有项目结构，尽量复用已有 UI 风格与常用组件。
2. 优先做“经典 2048 核心可玩版本”，不要把时间花在过度动画或大范围重构上。
3. 尽量实现：
   - 键盘方向键控制
   - 移动端触控/滑动控制，若实现滑动成本过高，至少提供清晰可用的方向按钮
4. 尽量控制改动范围在 2048 相关页面/组件与首页接入处，不要顺手改 unrelated 功能。
5. 若项目可执行 lint / build，请至少运行与本次改动相关的必要验证；如果 lint 因历史问题失败，要在报告中明确说明是新增还是存量问题。

## 建议实现方向
可参考项目内其他游戏页面的组织方式，按需新增：
- `app/2048/page.tsx`
- `components/game2048/*` 或等价目录
- `lib/game2048/*` 或等价逻辑文件

命名不强制，但结构应清晰、可维护。
