# TASK-2026-03-22-werewolf-sheriff-before-first-night-deaths

## 背景
classic-games-generator 项目的狼人杀白天流程存在一个时序问题。

## 用户反馈
第一天天亮后，应先进入“争警长”环节；只有在争警长结束之后，才能公布第一天晚上的死亡情况。
当前实现中，第一晚死亡信息暴露得过早，需要调整流程顺序。

## 任务目标
修复狼人杀首日白天流程时序：
- 第一天白天开始后，先进入争警长环节
- 争警长结束后，再公布首夜死亡信息
- 不要破坏后续白天发言、投票、结算等既有逻辑

## 执行要求
1. 先定位当前首日白天状态机/阶段推进逻辑，找出死亡信息在哪个阶段被过早暴露。
2. 只做与该流程问题直接相关、低风险的修复；不要顺手大改整个状态机。
3. 修复后自行做最小必要验证，至少覆盖：
   - 完成首夜流程并进入第一天白天
   - 先出现争警长环节
   - 争警长结束前，不公布首夜死亡情况
   - 争警长结束后，再公布首夜死亡情况
   - 后续白天流程仍可继续推进
4. 如果项目有 lint / build / 可执行测试，优先跑与本改动相关的最小验证。
